# Лабораторная работа 5: Моделирование Scrum

## Описание проекта
Консольное приложение для управления задачами. Позволяет создавать, просматривать, отмечать выполненные и удалять задачи. Данные сохраняются в файловой системе. Учебный проект для освоения Scrum методологии.

## Product Backlog
Создано 8 issues в репозитории. Каждая задача содержит заголовок, описание и критерии готовности. Задачи охватывают основные функции приложения и технические требования.

## Sprint 1 Planning
Выбрано 5 ключевых задач. Помечены меткой sprint-1, указаны приоритеты и оценка сложности. Каждая задача имеет цель и ожидаемый результат.

## Scrum-артефакты в Issues
Для каждой задачи указаны: описание, критерии готовности, приоритет, оценка сложности и принадлежность к спринту.

## Ход спринта
Прогресс отслеживался через изменение статусов. Часть задач переведена в состояние in-progress. Две задачи закрыты, одна осталась незавершенной.

## Sprint Review
**Закрытые задачи:** Создание базовой структуры проекта и реализация функции добавления задач.

**Доставленный инкремент:** Рабочее приложение с возможностью добавления задач, которые сохраняются в файл.

**Незавершенные задачи:** Отображение списка задач (частично), отметка выполненных и удаление задач (не начаты).

## Sprint Retrospective
**Хорошо:** Четкие критерии готовности, правильная оценка простых задач, регулярное обновление статусов.

**Не так:** Недооценена сложность работы с файлами, не хватило времени на обработку ошибок, слишком оптимистичное планирование.

**Улучшить:** Детальнее разбивать задачи, добавлять задачи по обработке исключений, проводить ежедневные встречи, создавать прототипы перед реализацией.
